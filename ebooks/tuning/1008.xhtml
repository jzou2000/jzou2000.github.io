Saving Profiling Data

The —save=xxx option for opcontrol allows the data to be saved for future reference. This allows the data from multiple experiments to be saved separately. It should be noted that there is a need to avoid modifying the executables. oprofile cannot perform the analysis if the executable has been changed. oprofile needs both the saved sample files and the unchanged executable.

Thus, for the chp10-profile1 example, the data could be saved with the following command:

opcontrol—save=chp10-profile1

The addition of session:chp10-profile1 specifies the session saved by this opcontrol. Thus, the commands used earlier to analyze chp10-profile1 would now be as follows:

opreport session:chp10-profile1—long-filenames

opreport session:chp10-profile1 -l image:/usr/src/chp10/chp10-profile1

opreport -l session:chp10-profile1 image:/usr/src/chp10/chp10-profile1—include-
symbols=Lookup_thread -details

opannotate session:chp10-profile1—source /usr/src/chp10-profile1
