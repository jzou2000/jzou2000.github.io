Uses of Kprobes

A Kprobe can be used to trigger a dump when a specific set of circumstances occurs. This is one of the most powerful aspects of Kprobes—being able to watch for a rare set of conditions and to get a snapshot of the system when they occur.

dpcc, the Linux Trace Toolkit (LTT), and Dprobes provide the infrastructure to do all-points-addressable dynamic tracing. This combination is a very powerful facility. You can easily generate a set of probe points from the nm output. From that, within minutes you can have a functional execution trace of a driver or module. Dynamic probes are aimed at solving the following problems:

  • Extreme debugging where problem re-creation cannot be forced

  • Highly available systems where debugging has to be done (for reasons of complexity you need to force a reproduction) on a production/live system

  • As an alternative to inserting a temporary printf or printk where rebuild time can be saved (this especially applies to kernel development)
