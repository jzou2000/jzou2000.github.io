Booting UML

Running the UML kernel (linux) program boots the user-mode kernel and brings up the system stored in the file root_fs. The UML looks for the root file system located in the current working directory, or the location of root_fs can be passed to the command line that loads the user-mode kernel. ubdx, where x is 0 to n, is a virtual disk for UML.

If gdb is required to debug a UML kernel, the debug parameter is included on the UML command line. So to start the UML kernel, the root image is root_fs.ext2 and the swap file is swap_fs. The UML command would look like this:

linux debug ubd0=root_fs.ext2 ubd1=swap_fs
