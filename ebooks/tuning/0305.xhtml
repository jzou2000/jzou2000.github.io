Debugging Using a Core File

For a user mode application to be able to create a core file, the system limit on the size of the core files must be greater than 0.

The default core file size is 0. When the core file size is 0, the system does not allow a core dump. Figure 3.6 shows that the core file size is 0 using the ulimit command with the -a option. To change the core file to 500,000, use the ulimit -c option.
